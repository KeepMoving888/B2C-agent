# 多智能体设计

## 5 大核心智能体

### 1. 咨询Agent (Consultation Agent)

**职责边界**：处理商品咨询、库存查询、技术支持等通用咨询问题。

**触发条件**：
- 意图 = 商品咨询 / 缺货询问 / 技术支持
- 不满情绪 < 阈值

**处理流程**：
1. 接收 RAG 检索的商品知识
2. 结合历史对话理解上下文
3. 生成专业、有温度的回复
4. 翻译为目标语言

**边界约束**：
- 仅处理商品信息类问题
- 涉及订单/物流/退款时建议转交对应 Agent

---

### 2. 订单Agent (Order Agent)

**职责边界**：处理订单查询、物流跟踪、催发货、地址修改等订单相关问题。

**触发条件**：
- 意图 = 物流查询 / 催发货 / 地址修改

**处理流程**：
1. 查询订单状态（对接订单系统，开源版本为规则回退）
2. 调用物流追踪接口
3. 处理地址修改请求
4. 生成回复并翻译

**边界约束**：
- 涉及退款/退换货时转交售后 Agent
- 复杂订单异常转人工

---

### 3. 售后Agent (Aftersales Agent)

**职责边界**：处理退款、退换货、商品损坏、错发漏发等售后问题。

**触发条件**：
- 意图 = 售后退款 / 退换货

**处理流程**：
1. 检索售后政策知识库
2. 判断售后类型（退款/换货/补发）
3. 生成解决方案并安抚情绪
4. 创建售后工单

**边界约束**：
- 投诉升级或情绪强烈时转人工
- 涉及法律纠纷转合规 Agent

---

### 4. 合规Agent (Compliance Agent)

**职责边界**：处理支付问题、隐私政策、数据合规、跨境贸易合规等问题。

**触发条件**：
- 意图 = 支付问题
- 涉及隐私/GDPR/关税咨询

**处理流程**：
1. 检索合规政策知识库
2. 严格基于政策回复，不超承诺
3. 涉及法律风险时建议转人工

**边界约束**：
- 严谨、合规、不承诺超出政策范围
- 法律纠纷必须转人工

---

### 5. 人工转接Agent (Human Handoff Agent)

**职责边界**：处理复杂问题、投诉升级、不满情绪客户、超出 AI 处理范围的问题。

**触发条件**：
- 不满情绪 ≥ 阈值（默认 60%）
- 意图 = 投诉处理
- 客户明确要求人工服务
- 复杂售后问题超出 AI 范围

**处理流程**：
1. 安抚客户情绪
2. 生成对话摘要，便于人工接手
3. 创建转接工单
4. 分配至对应技能组

**输出**：
- 对客户：安抚回复（目标语言）
- 对人工：对话摘要 + 客户画像 + 情感分析

---

## 条件路由规则

| 条件 | 目标 Agent | 说明 |
|------|-----------|------|
| 不满情绪 ≥ 60% | 人工转接Agent | 情绪阈值触发 |
| 意图 = 物流查询 / 催发货 / 地址修改 | 订单Agent | 订单类问题 |
| 意图 = 售后退款 / 退换货 | 售后Agent | 售后类问题 |
| 意图 = 支付问题 | 合规Agent | 合规类问题 |
| 意图 = 投诉处理 | 人工转接Agent | 投诉直接转人工 |
| 其他 | 咨询Agent | 默认兜底 |

## 多轮对话状态管理

通过 `AgentState` 解决长会话信息丢失与意图偏移问题：

- **历史消息保留**：保留最近 6 轮对话，提供上下文
- **意图槽位跟踪**：记录每轮意图，检测偏移
- **情感累积**：累积负面情绪，达到阈值触发转人工
- **RAG 上下文复用**：相似问题复用检索结果，减少重复检索
