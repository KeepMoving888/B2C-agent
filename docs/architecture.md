## 核心流程

### 1. 会话处理主流程

```
客服输入消息（中文）
        │
        ▼
  ┌─────────────┐
  │ 意图识别     │ ← 关键词规则 / 大模型双路
  │ 情感分析     │ ← 关键词规则 / 大模型双路
  └──────┬──────┘
         │
         ▼
  ┌─────────────┐
  │ RAG 检索     │ ← Milvus 向量检索 + BM25 关键词检索
  │ 重排序       │ ← Cross-Encoder 重排序
  └──────┬──────┘
         │
         ▼
  ┌─────────────────────┐
  │ 条件路由             │
  │ ┌─ 不满≥阈值 ──→ 人工转接Agent
  │ ├─ 物流/订单 ──→ 订单Agent
  │ ├─ 退款/退换 ──→ 售后Agent
  │ ├─ 支付/合规 ──→ 合规Agent
  │ └─ 其他      ──→ 咨询Agent
  └──────┬──────────────┘
         │
         ▼
  ┌─────────────┐
  │ Agent 处理   │ ← 结合 RAG 上下文 + 思维链 Prompt
  │ 生成中文回复 │
  └──────┬──────┘
         │
         ▼
  ┌─────────────┐
  │ 多语言翻译   │ ← 大模型翻译 / 关键词回退
  └──────┬──────┘
         │
         ▼
  返回前端（目标语言回复 + 中文回译 + 路由说明）
```

### 2. 长对话状态管理

通过 LangGraph 的 `AgentState` 管理多轮对话状态：

- **历史消息**：保留最近 6 轮对话，防止长会话信息丢失
- **意图槽位**：跟踪意图变化，检测用户意图偏移
- **情感状态**：累积情感分析，触发人工转接阈值
- **RAG 上下文**：跨轮复用检索结果，减少重复检索

## 模块职责

| 模块 | 职责 | 关键文件 |
|------|------|---------|
| 前端 | 三栏交互、多语言展示、实时通信 | `frontend/` |
| API 路由 | REST + WebSocket 接口 | `backend/app/api/` |
| 多智能体 | 5 大 Agent 编排与条件路由 | `backend/app/agents/` |
| RAG 检索 | 混合检索 + 重排序 + CoT | `backend/app/rag/` |
| 模型服务 | vLLM 对接、翻译、情感、意图 | `backend/app/services/` |
| 训练 | QLoRA 微调 + AWQ 量化 | `training/` |
| 部署 | Docker + vLLM + Milvus | `deployment/` |

## 降级策略

系统采用多级降级，保证服务可用性：

| 组件 | 在线模式 | 离线回退 |
|------|---------|---------|
| vLLM 推理 | 大模型生成回复 | 规则引擎 + 模板回复 |
| Milvus 检索 | 向量 + BM25 混合检索 | 内存关键词检索 |
| Embedding | bge-large-zh-v1.5 | 随机向量（仅保持流程） |
| 重排序 | bge-reranker Cross-Encoder | 关键词重叠度打分 |
| 翻译 | 大模型翻译 | 关键词替换 |

## 部署架构

### 开发环境

```
浏览器 ──→ 前端(8080) ──→ 后端(8000) ──→ vLLM(8001, 可选)
                                     └──→ Milvus(19530, 可选)
```

### Docker 部署

```
浏览器 ──→ Nginx(8080) ──→ FastAPI(8000) ──→ vLLM(8001, 可选)
          (前端静态)        (后端)           (Milvus 19530)
```
